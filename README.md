# AI-first development kit

Workflow and boilerplate to deliver MVPs in 2 days using Claude Code
with agents, worktrees, and auto mode.

---

## First-time setup (once per machine)

**1. Install Claude Code** ‚Äî [code.claude.com](https://code.claude.com)

**2. Enable auto mode**

Auto mode lets agents work unattended. Without it, agents pause for
approval on every action and the parallel workflow breaks down.

Open a terminal and start Claude Code:
```bash
claude
```

Press `Shift+Tab` repeatedly until the status bar shows **auto mode**.
You will see an opt-in prompt the first time ‚Äî accept it.

That's it. Claude Code remembers the setting for future sessions.

> Auto mode requires a Max plan and Claude Opus 4.6 or Sonnet 4.6+.

## Starting a new project

```bash
gh repo create my-project --template your-org/kit
cd my-project
claude          # Terminal 1 ‚Äî always keep this one open
```

The TL has already written `docs/PRD.md` and `docs/TRD.md`.

```
/init-project
```

Review and approve the three generated files (~10 min):
- `CLAUDE.md` ‚Äî stack and universal rules
- `docs/CONVENTIONS.md` ‚Äî project conventions
- `docs/PLAN.md` ‚Äî modules, dependencies, classification

Then launch the first wave:
```
/next-wave
```

## Dev workflow

**Terminal 1** is where you live. Keep it open the entire sprint.

```
/next-wave
‚Üí prints commands for the next wave of worktrees
```

Open one new terminal per printed command and run it:
```bash
# Terminal 2
claude --worktree wt-auth

# Terminal 3
claude --worktree wt-products
```

Agents work unattended. You only look at those terminals if one pauses.

When an agent finishes, merge its branch from **Terminal 1**:
```bash
git merge wt-auth
/next-wave        ‚Üê always run this right after a merge
```

`/next-wave` detects the merge, updates `docs/PLAN.md`, and prints
the next wave. Repeat until all modules are ‚úÖ.

---

## Terminal layout

| Terminal | What runs there |
|----------|----------------|
| **1** | Claude Code on `main`. `/init-project`, `/next-wave`, merges. |
| **2, 3, 4...** | One worktree per module, unattended in auto mode. |

3 parallel modules ‚Üí 4 terminals total.
5 parallel modules ‚Üí 6 terminals total.

**tmux (optional):** `/next-wave` prints a tmux one-liner when detected.
Available on macOS (`brew install tmux`) and Linux (pre-installed).
On Windows: use WSL2 or open terminal tabs manually.

## Auto-learning ‚Äî where each thing goes

| What the agent learned | Destination |
|------------------------|-------------|
| Decision affecting only its module | AI Notes in `docs/PLAN.md` |
| Convention other modules should follow | `docs/CONVENTIONS.md` |
| Stack pitfall that affects future projects | Root `CLAUDE.md` |

---

## Rules

1. **Auto mode** ‚Äî enable once via `Shift+Tab` in the CLI before launching worktrees
2. **After every merge** ‚Äî run `/next-wave` immediately from Terminal 1
3. **Migrations** ‚Äî no agent runs them; blocked in `.claude/settings.json`
4. **PRD and TRD** ‚Äî write-protected during active development
5. **Max 5 worktrees** in parallel ‚Äî more exceeds review capacity
6. **Main always builds** ‚Äî `/next-wave` checks before printing commands
